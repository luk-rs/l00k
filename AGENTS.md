## Project overview

**l00k** is a modular TradingView Pine Script v6 custom indicator.
Source modules live as individual `.pine` files in `src/` and are concatenated
into a single unified script (`dist/l00k.pine`) by CI on every push to `main`.

The unified script is published to a GitHub Gist so the URL never changes.

## Architecture

```
l00k/
├── src/                        # Pine Script v6 source modules
│   ├── 00_meta.pine            # //@version=6 + indicator() — ALWAYS first
│   ├── [10-19]_*.pine          # types & constants
│   ├── [20-29]_*.pine          # shared inputs
│   ├── [30-49]_*.pine          # shared utility functions
│   ├── [50-79]_*.pine          # indicator / strategy vertical slices
│   ├── [80-89]_*.pine          # alerts
│   └── [90-99]_*.pine          # shared plots & visuals
├── scripts/
│   ├── build.sh                # concat src/*.pine → dist/l00k.pine
│   └── changelog.sh            # generate CHANGELOG.md from tags
├── .github/workflows/
│   └── publish.yml             # CI: version → build → gist → tag → changelog
├── .gist_id                    # auto-managed by CI (gist identifier)
├── CHANGELOG.md                # auto-generated by CI
└── dist/                       # build output (gitignored)
```

## Key rules

### Pine Script conventions
- Target: **Pine Script v6** (`//@version=6`).
- The `//@version=6` directive and `indicator()` call MUST only appear in
  `src/00_meta.pine`. No other file should contain these.
- Files are concatenated in **lexicographic order** by filename. Use numeric
  prefixes to control ordering (see table above).
- Prefer self-contained vertical slices in the `50-79` range for indicators
  and strategies. A slice may include its own inputs, helpers, calculations,
  plots, lines, labels, and alerts when those symbols are not reused elsewhere.
- Use `10-49` and `90-99` only for truly shared types, constants, inputs,
  helpers, plots, and visuals.
- All Pine Script code must be valid when concatenated. Ensure no duplicate
  variable/function names across files.

### Commit conventions
This project uses **Conventional Commits** for automatic semantic versioning:
- `feat:` → minor bump
- `fix:` → patch bump
- `feat!:` or `BREAKING CHANGE` → major bump
- Anything else (`chore:`, `docs:`, `refactor:`, etc.) → patch bump

Always use conventional commit prefixes. Scope is optional: `feat(rsi): ...`

### CI pipeline
- Triggers on push to `main` when `src/**` or `scripts/**` change.
- Do NOT manually edit `CHANGELOG.md`, `.gist_id`, or `dist/`.
- Do NOT create version tags manually — CI manages `vX.Y.Z` tags.
- The `GIST_TOKEN` secret (classic PAT with `gist` scope) must exist.

### File management
- New indicators go in `src/` with an appropriate numeric prefix. Prefer the
  next available `50-79` prefix for a self-contained vertical slice.
- When adding a new file, verify it doesn't break concatenation order
  (e.g. a plot referenced before it's defined).
- When removing a file, check that no other file depends on symbols it defines.
